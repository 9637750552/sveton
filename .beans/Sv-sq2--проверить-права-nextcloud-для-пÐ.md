Проверить, можно ли в текущем Nextcloud выдать монтажнику доступ к папке объекта с чтением и загрузкой/созданием без удаления.

- [x] Проверить версию Nextcloud и приложение files_sharing.
- [x] Проверить поддерживаемые permission bits для шаринга.
- [x] Проверить, есть ли режим read+create/upload без delete.
- [x] Зафиксировать практический вывод для папок объектов.

## Summary of Changes

Проверена установка Nextcloud 34.0.1: files_sharing включен, права шаринга представлены битами READ=1, CREATE=4, UPDATE=2, DELETE=8, SHARE=16. Комбинация READ+CREATE имеет маску 5 и валидна для папочных ссылок/шаринга; публичная загрузка не запрещена настройками, значит действует дефолт yes. Практический вывод: папки объектов нужно создавать с явной маской 5 для монтажника, а не с дефолтными полными правами.
