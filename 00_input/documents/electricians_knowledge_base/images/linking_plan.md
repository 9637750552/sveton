# План привязки изображений к знаниям

Дата обновления: 2026-06-16

## Назначение

Этот файл задает правила, по которым изображения из `inventory.csv` будут связываться с будущими чанками текста и атомарными утверждениями.

Главное правило: изображение не создает техническое утверждение само по себе. Оно может быть привязано к утверждению только если утверждение подтверждено текстом источника, а изображение является иллюстрацией или схемой к этому же фрагменту.

## Поля для связывания

В `inventory.csv` добавлены поля:

- `media_ref`: исходная ссылка вида `media/imageN.ext` или `page_NNN`;
- `source_anchor`: ближайший текстовый якорь в документе;
- `nearby_text`: очищенный текст вокруг картинки;
- `linking_bucket`: решение для следующего этапа связывания;
- `related_section`: будущий раздел базы знаний.

В атомарных утверждениях используем поле:

- `related_image_ids`: список `image_id`, которые иллюстрируют или уточняют утверждение.

## Linking Buckets

- `ready_for_linking`: можно использовать на этапе chunking/semantic extraction без дополнительного отбора;
- `candidate_for_linking`: можно предлагать как связанную картинку, но финальная привязка требует проверки nearby text;
- `manual_pdf_review`: PDF-страница отрендерена как изображение, но page-level текст пока ненадежен;
- `exclude_candidate`: мелкие иконки, служебные элементы, декоративные или неясные фрагменты.

## Правила для semantic extraction

1. Если чанк содержит `media_ref`, связать его с соответствующим `image_id` из `inventory.csv`.
2. Если атомарное утверждение извлечено из текста рядом с картинкой, добавить `image_id` в `related_image_ids`.
3. Если рядом несколько картинок, добавлять только те, которые подтверждаются `nearby_text` или явно относятся к одному примеру/схеме.
4. Для `manual_pdf_review` не ставить `related_image_ids` автоматически. Сначала нужен ручной просмотр страницы.
5. Для `exclude_candidate` не ставить `related_image_ids`, кроме случаев ручного переопределения.
6. Не переносить подпись картинки в атомарное утверждение без текстового подтверждения в источнике.

## Приоритеты ручного отбора

1. `photo_report`: фото-примеры правил съемки монтажа. Статус: выполнено для `img_0001`, `img_0002`; связь сохранена в `statement_images.jsonl`.
2. `basic_knowledge`: схема последовательного и параллельного соединения АКБ.
3. `installation_process`: схемы установки АКБ, защиты DC, байпаса, реверсивной трассы, выделения резервной группы.
4. `ups_components`: схемы состава ИБП, балансира, выравнивающих перемычек, DC-защиты и техкарты изготовления кабелей/перемычек.
5. `distribution_boards`: схема щита клиента, УЗО, контакторы, мастер-выключатель, сохранение логики работы при выделении резерва.

## Ограничение

Автоматическая привязка контекста выполнена по DOCX markdown-источникам через `media/imageN`. Для PDF пока создана только page-level визуальная база. PDF-страницы нужно отдельно просмотреть перед использованием в базе знаний.
