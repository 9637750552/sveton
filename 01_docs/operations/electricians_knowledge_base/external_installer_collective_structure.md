# Структура Collective для внешних монтажников

Дата: 2026-06-29

Статус: рабочая карта сборки экспериментального Collective.

Назначение: определить, как должна выглядеть отдельная база знаний для внешних электриков-партнеров DOMIBP и как собирать ее страницы из бизнес-процесса и атомарных утверждений Postgres-БЗ.

## Принцип сборки

Новый Collective не должен копировать старые внутренние инструкции один к одному.

Сборка идет в таком порядке:

1. Бизнес-процесс задает последовательность работы по объекту.
2. Атомарные утверждения Postgres-БЗ дают факты, требования, чек-листы и технические правила.
3. Старые DOCX/PDF остаются первоисточниками и приложениями, но не определяют структуру нового раздела.
4. Каждая будущая страница Collective должна иметь внутреннюю трассировку к `statement_id`.

Для этой экспериментальной версии текущие поля ревью в Postgres не ограничивают проектирование структуры. Предполагаем, что все утверждения валидны и могут использоваться для сборки.

## Что должно быть на каждой рабочей странице

Для страниц `03-13` использовать единый формат:

```text
Цель этапа
Когда выполнять
Что сделать
Что проверить
Что сфотографировать / снять на видео
Что отправить в Светон
Когда остановиться и позвать инженера
Ошибки, из-за которых работу не примут
Документы и примеры
Внутренние statement_id
```

Монтажнику во внешнем Collective можно не показывать `statement_id` в основном тексте. Но в исходном Markdown перед переносом в Nextcloud их нужно сохранять в служебном блоке или комментарии, чтобы не потерять трассировку.

## Как использовать кластеры при сборке

### Процессные страницы

Страницы `02-13` должны идти по бизнес-процессу, а не по старым названиям файлов.

Основные источники:

- `C002` для обязанностей, закрытия выезда, передачи результата;
- `C003` для отчета и фиксируемых параметров;
- `C004` для фото/видео;
- `C005` для входных материалов, ТЗ и папки объекта;
- `C009` для монтажа, настройки, проверки и обучения клиента;
- `C011` для распределения ролей на объекте;
- `C012` для УЗО;
- `C014` для технических карт изделий.

### Справочные страницы

Страница `15 - Технический справочник` не должна быть первым экраном для монтажника. Это поддерживающий раздел, куда монтажник переходит из конкретного шага.

Основные источники:

- `C006` для базовых понятий;
- `C007` для состава ИБП;
- `C008` для щитов, УЗО, шин и мастер-выключателя;
- `C012` для отдельной темы УЗО;
- `C014` для технических карт.

### Документы и шаблоны

Страница `16 - Документы, шаблоны и примеры` должна хранить файлы для просмотра и скачивания, но не заменять пошаговые инструкции.

Минимальный набор:

- шаблон отчета по монтажу;
- чек-лист фотофиксации;
- пример правильного отчета;
- исходные инструкции PDF/DOCX;
- схемы и диаграммы процесса;
- список того, что отправлять после первого и второго выезда.

## Важное отличие от старых внутренних инструкций

Старые инструкции были написаны для внутреннего монтажника, который получает задачу от начальника монтажного отдела и в основном выполняет монтаж.

Новая база пишется для внешнего профессионального электрика-партнера. Поэтому она должна явно покрывать:

- самостоятельное согласование первого выезда с клиентом;
- обследование объекта до подтверждения работ;
- проверку щита, места установки и трассы;
- замеры нагрузок и фиксацию расхождений;
- подтверждение ТЗ с клиентом;
- решение, делать ли подготовку на первом выезде или переносить ее на второй;
- самостоятельную ответственность за подготовительные и электромонтажные работы;
- удаленную консультацию и приемку Светон;
- отчетность, фото/видео и рейтинг партнера.

## Следующий шаг

После утверждения этой структуры нужно собрать первые две страницы:

1. `00 - Старт`
2. `01 - Контакты, доступ и связь`
3. `03 - Как устроен процесс по объекту`

Они зададут язык и формат всего Collective. После этого можно собрать блок первого выезда из `C002`, `C003`, `C004`, `C005`, `C008` и бизнес-процесса.
